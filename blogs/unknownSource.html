Tech Log:"Unknown Source" in java debug

wangxp.site

2017.9.6 by wangxp

前几天遇到一个bug（过程大家可以忽略，直接看结论），只出现在正式版本上，测试版本一直没出现问题。因为打印了stackTree，所以能够看到报错是某一个具体方法 的问题，但是这个stackTree就仅仅只是追踪到该错误方法外，显示xxx.xxx.function(Unknown Source)。 最终显示的异常是java.lang.NullPointerException。由于当时不是特别了解项目类中仅仅加一个set方法就能够调用其他类的实现逻辑，所以我 根据后面报的reflict反射异常一直以为是spring里面的配置哪出错了。然后开始将报错信息分开复制出来百度，百度了半天也没结果（不得不说百度 上关于英文相关的专业参考资料还是蛮少的），后面实在没办法了就用Google搜索到了unknown source的一点东西。最后看到有人建议在build.xml设置debug=true， 而刚刚好我们的项目也需要build一个基础包，于是默默地修改代码，提交上去，等到正式版本发布了，异常就准确的显示了基础包里面的某个方法错误。well done:)

结论：

在需要生成基础jar包的时候，如果可以，尽量在build.xml里设置好debug=true这一属性。这样使得程序报异常的时候能够准确追踪异常代码，
